odsTimerange
directive in module ods-widgets

Description

This widget displays two fields to select the two bounds of a date and time range.

Usage

as element:
<ods-timerange
       context="{DatasetContext|DatasetContext[]}"
       time-field="{string}"
       default-from="{string}"
       default-to="{string}"
       display-time="{string}"
       date-format="{string}"
       suffix="{string}"
       label-from="{string}"
       label-to="{string}"
       placeholder-from="{string}"
       placeholder-to="{string}"
       to="{string}"
       from="{string}">
</ods-timerange>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
contextDatasetContextDatasetContext[]

Dataset Context or array of context to use

timeFieldstring

Name of the field (date or datetime) to filter on

(default: first date/datetime field available)

defaultFromstring

Default datetime for the "from" field: either "yesterday", "now" or a string representing a date

(default: none)

defaultTostring

Default datetime for the "to" field: either "yesterday", "now" or a string representing a date

(default: none)

displayTimestring

Define if the date selector displays the time selector as well

(default: true)

dateFormatstring

Define the format for the date displayed in the inputs

(default: 'YYYY-MM-DD HH:mm')

suffixstring

(optional) Add a suffix to the q.timerange, q.from_date or q.to_date parameter. This prevents widgets from overriding each other.

(default: 'fieldname')

labelFromstring

Set the label before the first input

(default: 'From')

labelTostring

Set the label before the second input

(default: 'to')

placeholderFromstring

Set the label before the first input

(default: '')

placeholderTostring

Set the label before the second input

(default: '')

tostring

Set a variable that will get the iso formatted value of the first input

(default: none)

fromstring

Set a variable that will get the iso formatted value of the second input

(default: none)
